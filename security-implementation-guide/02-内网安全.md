## 1. VPC 网络架构

```
┌─────────────────────────────────────────────────────────────┐
│  AWS VPC（10.0.0.0/16）                                      │
│                                                              │
│  ┌──────────────────┐  ┌──────────────────┐                 │
│  │  公有子网         │  │  公有子网         │  （多可用区）   │
│  │  10.0.1.0/24     │  │  10.0.2.0/24     │                 │
│  │  ALB / NAT 网关  │  │  ALB / NAT 网关  │                 │
│  └────────┬─────────┘  └────────┬─────────┘                 │
│           │                     │                           │
│  ┌────────▼─────────────────────▼─────────┐                 │
│  │  私有子网  10.0.10.0/24                 │                 │
│  │  K8s 工作节点 / 微服务                  │                 │
│  │  API Gateway / Traefik Ingress         │                 │
│  └────────────────────┬───────────────────┘                 │
│                       │                                     │
│  ┌────────────────────▼───────────────────┐                 │
│  │  数据子网  10.0.20.0/24                 │                 │
│  │  RDS / ElastiCache / Kafka             │                 │
│  │  禁止直接访问公网                       │                 │
│  └────────────────────┬───────────────────┘                 │
│                       │                                     │
│  ┌────────────────────▼───────────────────┐                 │
│  │  管理子网  10.0.30.0/24                 │                 │
│  │  SIEM / Vault / 堡垒机                  │                 │
│  │  VPN 终端 / 跳板服务器                  │                 │
│  └────────────────────────────────────────┘                 │
└─────────────────────────────────────────────────────────────┘
```

**核心规则：**
→ 公有子网：只放 ALB，不放应用服务器
→ 私有子网：禁止直接访问公网，出站流量走 NAT 网关
→ 数据子网：完全隔离，不能访问公网，只接受私有子网的连接
→ 管理子网：只能通过 VPN 或堡垒机访问，要求 MFA

## 2. 安全组规则（关键配置）

**ALB 安全组**
```
入站：443（HTTPS）来自 0.0.0.0/0
      80（HTTP）来自 0.0.0.0/0 → 强制跳转到 443
出站：8080 仅允许到私有子网安全组
```

**API Gateway / Traefik 安全组**
```
入站：8080 仅来自 ALB 安全组
      8443 仅来自 ALB 安全组
出站：所有流量到应用服务安全组
```

**应用服务安全组**
```
入站：8080-9090（应用端口）仅来自 API 网关安全组
      15000-15010（Istio Sidecar）仅来自私有子网内部
出站：5432（PostgreSQL）到数据安全组
      6379（Redis）到数据安全组
      9092（Kafka）到数据安全组
```

**数据子网安全组**
```
入站：5432 仅来自应用安全组（PostgreSQL）
      6379 仅来自应用安全组（Redis）
      9092 仅来自应用安全组（Kafka）
      禁止所有来自 0.0.0.0/0 的入站
出站：受限，禁止访问公网
```

**管理子网安全组**
```
入站：22（SSH）仅来自 VPN 网段（10.0.31.0/24）
      443 仅来自 VPN 网段
      拒绝所有来自 0.0.0.0/0 的入站
出站：全放通（监控需要较广的出站权限）
```

## 3. API 命名空间隔离 【已实施 — 事故后修复】

关键原则：`/internal/` 命名空间绝对不能被公网访问。

**Traefik IngressRoute 配置：**
```yaml
# 对外路由 — 通过 CloudFront → ALB → API 网关暴露
apiVersion: traefik.io/v1alpha1
kind: IngressRoute
metadata:
  name: 公开路由
spec:
  entryPoints:
    - websecure          # 监听 443
  routes:
    - match: PathPrefix(`/user/public/`) || PathPrefix(`/payment/public/`)
      kind: Rule
      services:
        - name: api-gateway-service
          port: 8080
    # /internal/ 路径完全不在此处配置

---
# 内部路由 — 仅绑定内网 IP，外网无法访问
apiVersion: traefik.io/v1alpha1
kind: IngressRoute
metadata:
  name: 内部路由
  namespace: internal
spec:
  entryPoints:
    - internal-only   # 仅绑定 10.0.10.x，不绑定 0.0.0.0
  routes:
    - match: PathPrefix(`/internal/`)
      kind: Rule
      middlewares:
        - name: internal-token-auth   # 强制校验 X-Internal-Token
      services:
        - name: internal-service
          port: 8081
```

## 4. 服务间通信 — mTLS（Istio）【计划中】

所有微服务之间的通信必须使用双向 TLS 认证。

**全局强制 mTLS（集群级别）：**
```yaml
apiVersion: security.istio.io/v1beta1
kind: PeerAuthentication
metadata:
  name: default
  namespace: istio-system
spec:
  mtls:
    mode: STRICT   # 拒绝所有非 mTLS 流量
```

**服务间访问白名单（授权策略）：**
```yaml
apiVersion: security.istio.io/v1beta1
kind: AuthorizationPolicy
metadata:
  name: 卡服务访问策略
  namespace: production
spec:
  selector:
    matchLabels:
      app: card-service
  rules:
    - from:
        - source:
            principals:
              - "cluster.local/ns/production/sa/subscription-service"
              - "cluster.local/ns/production/sa/kyc-service"
      to:
        - operation:
            methods: ["POST", "GET"]
            paths: ["/v1/cards/*"]
```

**mTLS 的价值：**
→ 每个服务通过证书证明身份，无法伪装
→ 即使某台服务器被攻破，攻击者也无法冒充其他服务
→ 所有服务间流量自动加密
→ 完整的服务间调用审计链路

## 5. 网络流量路径

```
客户端请求完整路径：

公网
  ↓ HTTPS
Cloudflare WAF（DDoS 过滤、Bot 拦截、OWASP 规则）
  ↓ HTTPS（Cloudflare → AWS 源站）
AWS CloudFront（CDN、缓存、TLS 终止）
  ↓ HTTPS
AWS WAF（OWASP CRS、限速、IP 信誉）
  ↓ HTTP（内部，SSL 已终止）
Application Load Balancer（健康检查、按域名路由）
  ↓ HTTP/2
Traefik Ingress Controller（K8s，按路径前缀路由）
  ↓ gRPC / HTTP（经 Istio Sidecar mTLS）
目标微服务
  ↓ TLS
数据层（RDS / Redis / Kafka）

出站流量路径（微服务 → 外部）：
微服务 → NAT 网关 → 公网
（适用于：Sumsub API、区块链 RPC、支付网关）
出站 IP 固定，在外部提供商处加入白名单
```

## 6. 办公室 / 远程访问

**VPN / ZTNA（SealSuite）：**
→ 当前：OpenVPN / WireGuard 接入管理子网
→ 目标：SealSuite ZTNA 替代 VPN
→ 所有 VPN / ZTNA 连接强制 MFA
→ 禁止分离隧道（Split Tunneling），全部流量走企业网关

**堡垒机（紧急直接访问）：**
→ 位于管理子网
→ 无永久 SSH 密钥，使用 Vault 签发基于会话的短期凭证
→ 所有会话录屏（终端录像存入 S3）
→ 不活跃超过 8 小时自动断开
