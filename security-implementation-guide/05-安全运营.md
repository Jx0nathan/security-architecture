## 1. SIEM 架构（Elastic Security）

**必须接入的日志来源：**
```
网络层：
→ Cloudflare WAF 日志（所有请求 + 处置决策）
→ AWS WAF 日志
→ VPC 流日志（Flow Logs）
→ ALB 访问日志

应用层：
→ API Gateway 访问日志（含用户 ID、IP、接口路径、状态码）
→ 应用错误日志（结构化 JSON 格式，不含 PII）
→ K8s 审计日志
→ Istio 访问日志（服务间调用记录）

数据层：
→ RDS 审计日志（所有对敏感表的查询）
→ Redis 访问日志
→ S3 访问日志（尤其是 KYC 存储桶）

身份层：
→ SSO 登录事件（成功 + 失败）
→ AWS CloudTrail（所有 API 调用记录）
→ Vault 审计日志（所有密钥访问记录）
→ SealSuite 端点事件

安全工具层：
→ SealSuite EDR 告警
→ AWS Security Hub 发现项
→ CSPM 告警

日志保留策略：
→ 热数据（可查询）：90 天（Elasticsearch）
→ 冷归档（S3）：合计保留 3 年
→ 日志不可修改（Immutable）
→ 含 PII 的日志：写入前必须脱敏
```

## 2. 关键告警规则

**P1 级别 — 立即响应（15 分钟内）**

```
规则名：API 批量爬取
触发条件：单个 IP 在 10 分钟内对同一接口请求超过 1000 次
处置措施：Cloudflare 自动封禁 + PagerDuty 告警

规则名：内部接口公网访问
触发条件：/internal/ 接口收到来自公网 IP 且 HTTP 状态码为 200 的响应
处置措施：立即封禁 + PagerDuty 告警 + 通知 CISO
【备注：这是 2026 年 2 月事故的核心告警规则，必须首先配置】

规则名：批量数据导出
触发条件：单用户/会话在 1 小时内获取超过 500 条不同用户记录
处置措施：自动挂起会话 + PagerDuty 告警

规则名：非工作时间特权访问
触发条件：管理员/root 账号在 23:00-07:00（新加坡时间）从异常 IP 登录
处置措施：强制 MFA 重新认证 + 安全团队告警

规则名：新建管理员账号
触发条件：创建新的 IAM 管理员角色或新建管理员账号
处置措施：通知 CISO + 要求 1 小时内确认授权
```

**P2 级别 — 2 小时内响应**

```
规则名：注入探测攻击
触发条件：WAF 在 1 小时内拦截同一 IP 超过 20 次注入攻击
处置措施：延长封禁至 24 小时 + 告警 SOC 分析师

规则名：认证失败频发
触发条件：同一账号 5 分钟内登录失败超过 10 次
处置措施：临时锁定账号 + 发送告警

规则名：异常大文件下载
触发条件：用户单次会话从内部系统下载超过 100MB
处置措施：告警 SOC 审查

规则名：证书即将过期
触发条件：任意 TLS 证书距过期不足 14 天
处置措施：告警 DevOps + 安全团队
```

## 3. SOC 团队结构与升级流程

```
L1 — 告警分诊（7×24 值班）
→ 审查 SIEM 告警
→ 分类：真实事件 / 误报 / 待深入调查
→ P1 告警 15 分钟内升级至 L2
→ 响应 SLA：P1 15 分钟，P2 2 小时，P3 下一工作日

L2 — 事件调查
→ 深入分析已确认事件
→ 协调系统负责人进行隔离处置
→ 保全证据（日志、内存镜像）
→ 事件扩大时升级至 L3

L3 — 事件指挥官（CISO / 高级安全负责人）
→ 重大事件统筹管理
→ MAS 通知决策
→ 对接外部机构（Blackpanda、MAS）
→ 对执行层和董事会进行通报
```

## 4. 事件响应 SOP（符合 MAS TRM §14）

```
T+0:00   事件发现
         → SIEM 告警触发 / 用户报告 / 外部通知
         → L1 值班分析师收到告警

T+0:15   初步研判
         → L1 评估严重等级
         → P1 级别：升级 L2 + 通知 L3（CISO）
         → 立即开启事件工单（必须，证据链）

T+1:00   ⚠️ MAS 通知截止时间（重大事件）
         → L3 判断：是否需要上报 MAS？
         重大事件判断标准：
           · 核心系统不可用超过 30 分钟
           · 任何客户数据泄露（不限规模）
           · 未经授权的系统访问
           · 影响客户资金的欺诈事件
         → 确认需上报：CISO 立即致电 MAS
         → 记录：致电人、时间、通话内容摘要

T+4:00   隔离处置
         → 隔离受影响系统
         → 保全证据（快照、日志）
         → 实施临时修复措施

T+24:00  ⚠️ MAS 书面报告截止时间
         → 提交初步书面报告至 MAS
         → 内容：事件时间线、影响范围、已采取的处置措施

T+持续   每日向 MAS 更新进展，直至事件关闭

T+30天   事件关闭后完整报告提交 MAS
（事件  → 完整根因分析（RCA）
关闭后） → 永久性整改方案
         → 经验教训总结
         → 已实施的控制改进措施
```

## 5. EDR / SealSuite 配置规范

**监控范围：**
→ 所有企业设备（笔记本电脑、工作站）
→ 可安装 Agent 的服务器端点

**必须配置的 SealSuite 告警规则：**
```
→ USB 设备接入：告警 + 数据复制需审批
→ 在敏感应用（交易系统、卡管理后台）进行截图：阻断
→ 向个人云存储（Google Drive、Dropbox）上传文件：阻断
→ 通过邮件发送大文件（>50MB）：触发审查
→ 安装未经审批的应用程序：告警
→ 设备策略合规性不通过：从网络隔离
```

## 6. CSPM — AWS Security Hub

```
→ AWS 基础安全最佳实践
→ CIS AWS Foundations Benchmark
→ PCI DSS v3.2.1

以下发现项必须在 24 小时内修复：
→ S3 存储桶公开访问
→ 安全组对 0.0.0.0/0 开放 22、3306 或 5432 端口
```
