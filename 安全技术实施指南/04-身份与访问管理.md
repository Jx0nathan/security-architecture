# 04 — 身份与访问管理（IAM）
**对应 MAS TRM §10 — 访问控制**
当前状态：部分已实施

---

## 1. 角色权限层级

```
超级管理员（仅限 CISO / CTO）
  ├── 生产管理员     → 完整生产环境访问，所有变更必须有工单
  ├── 安全管理员     → 安全工具、WAF、SIEM 配置权限
  └── 合规只读       → 只读审计访问权限

团队负责人
  ├── 开发工程师     → 仅限开发/测试环境，默认无生产环境权限
  ├── DevOps / SRE  → 生产只读 + 通过 CI/CD 流水线部署（不能直接改配置）
  └── 运营人员       → 生产只读 + 特定操作命令权限

只读权限
  └── 分析师 / 客服  → 脱敏数据，不可见原始 PII
```

## 2. MFA 强制要求

**以下场景必须开启 MFA（MAS TRM §10.2 要求）：**

```
必须 MFA 的场景：
✅ 所有生产系统访问
✅ 远程访问（VPN / SealSuite ZTNA）
✅ 所有特权账号（admin / root）
✅ 代码仓库（生产分支的 Merge 操作）
✅ 云控制台（AWS Console）
✅ 安全工具（SIEM、WAF 管理后台）
✅ 任何远程访问方式

不允许的 MFA 方式：
❌ 短信 OTP 不允许用于特权账号（存在 SIM Swap 风险）

推荐的 MFA 方式：
→ 硬件密钥（YubiKey）— CISO、CTO、生产 Admin 必须使用
→ TOTP 认证器（Google Authenticator）— 普通员工最低标准
```

## 3. API 认证规范

**对外 API（/public/ 命名空间）：**
```
认证方式：OAuth 2.0 + JWT
Access Token 有效期：15 分钟
Refresh Token 有效期：7 天
Token 存储方式：httpOnly Cookie（禁止存储在 localStorage）
权限范围：每个接口按 scope 进行校验
```

**内部 API（/internal/ 命名空间）：**
```
认证方式：内部服务 Token（X-Internal-Token 请求头）
         + 来源 IP 校验（必须在私有子网 CIDR 范围内）
Token 管理：由 Vault 生成，每 24 小时自动轮换
绝对不暴露到公网
```

**服务间调用：**
```
认证方式：mTLS（通过 Istio）
证书：Istio CA 自动签发，90 天轮换
身份：基于 SPIFFE/SPIRE 服务身份（不依赖 IP）
```

**管理 / 后台 API：**
```
认证方式：OAuth 2.0 + 敏感操作触发 MFA 二次验证
所有管理操作记录操作者身份日志
```

## 4. 特权访问管理（PAM）

**核心原则：**
→ 禁止共享管理员凭证，每个管理员有独立身份
→ 禁止持久特权会话，采用即时授权（Just-in-Time）
→ 所有特权会话录屏（终端录像存入 S3）
→ 应急账号（Break-glass）仅在紧急情况下使用，需两人授权

**HashiCorp Vault 实现方案：**
```
数据库凭证：
→ 动态凭证：Vault 为每次请求生成唯一用户名/密码
→ 凭证有效期：开发人员 1 小时，自动化流水线 15 分钟
→ 会话结束后自动撤销

SSH / 服务器访问：
→ Vault SSH CA 签发临时证书
→ 证书有效期：仅 1 小时
→ 服务器上禁止存储永久 SSH 密钥

密钥管理：
→ 所有 API 密钥、数据库密码、证书存入 Vault
→ 应用通过 Vault Agent 在启动时动态获取密钥
→ 代码、配置文件、CI/CD 日志中禁止出现任何密钥明文
```

## 5. AWS IAM 最小权限原则

```
规则：
→ 生产 IAM 策略中禁止使用通配符（*）权限
→ 开发 / 测试 / 生产使用独立 AWS 账号
→ 人类账号禁止使用长期 AWS Access Key
  （统一使用 IAM Role + 实例配置 / OIDC 联合认证）
→ 所有 IAM 变更通过 IaC（Terraform）管理
→ AWS Console 访问强制 MFA

必须配置的边界策略：
→ 禁止：CreateAccessKey（防止创建长期访问密钥）
→ 禁止：未开启 MFA 时执行任何操作（人类账号）
→ 禁止：在 ap-southeast-1 以外的地区执行操作（地区锁定）
```

## 6. 访问权限定期审查（MAS TRM §10）

```
审查频率：每季度一次（每 3 个月）
审查范围：
→ 所有生产系统的访问权限
→ 特权账号清单
→ 服务账号清单
→ AWS IAM 角色分配

审查流程：
→ 系统负责人审查并确认各自的用户列表
→ CISO 审批所有特权访问
→ 90 天内无活动的账号 → 自动禁用
→ 离职员工账号 → HR 通知后 4 小时内禁用
```

```
  □ 禁用 AWS IAM 访问权限
  □ 吊销其名下所有 API 密钥

  □ 收回企业设备
  □ 设备未归还时执行远程擦除
  □ 归档邮箱
  □ 确认所有会话已终止
```
